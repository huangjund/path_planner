\doxysubsection{Detailed Description}
Optimal Rapidly-\/exploring Random Trees Maintaining A Pseudo Optimal Tree. 

\label{classompl_1_1geometric_1_1RRTXstatic_gRRTXstatic}%
\Hypertarget{classompl_1_1geometric_1_1RRTXstatic_gRRTXstatic}%
\begin{DoxyParagraph}{Short description}
\mbox{\hyperlink{classompl_1_1geometric_1_1RRTXstatic_gRRTXstatic}{R\+R\+T\+Xstatic}} is an asymptotically-\/optimal incremental sampling-\/based motion planning algorithm. It differs from the R\+R\+T$\ast$ algorithm by maintaining a pseudo-\/optimal tree.~\newline
When adding a new motion, any rewiring improving the cost by more than epsilon is done. While each iteration is more costly than one of R\+R\+T$\ast$, the convergence is faster thanks to the pseudo-\/optimal tree.~\newline
Parameters are\+:
\begin{DoxyItemize}
\item range\+: maximum size allowed for an edge
\item goal\+\_\+bias\+: probability of sampling the goal region
\item rewire\+\_\+factor\+: multiplicative factor for the size of the neighbor ball (or the number of neighbors)
\item use\+\_\+k\+\_\+nearest\+: use k\+NN instead of r\+NN
\item informed\+\_\+sampling\+: direct sampling in the relevant region defined by the heuristic
\item sample\+\_\+rejection\+: rejects sampled out of the relevant region defined by the heuristic
\item number\+\_\+sampling\+\_\+attempts\+: number of sampling attempts when using rejection sampling or informed sampling
\item epsilon\+: the minimum threshold for cost improvement required to rewire the tree
\item update\+\_\+children\+: Force propagation of cost to children even if it is less than the threshold epsilon. It improves convergence with minimal extra computation.
\item rejection\+\_\+variant\+: Variants of the rejection of samples, 0\+: no rejection, 1-\/3\+: variants from reference (3)
\item rejection\+\_\+variant\+\_\+alpha\+: Parameter alpha used for the rejection sampling, it allows to scale the non-\/admissible heuristic, see reference (3)
\end{DoxyItemize}
\end{DoxyParagraph}
The queue is implemented only using a scalar (cost + heuristic) as a key for ordering. With random samples, the set \{cost + heuristic = constant\} should be of measure 0, so a more complex key is not needed.

\begin{DoxyParagraph}{Disclaimer}
Only the static part of the R\+R\+TX algorithm is implemented. Dynamical obstacles and updates of the robot position are not available in this implementation.
